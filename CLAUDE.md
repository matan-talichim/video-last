# AI Editor — CLAUDE.md (חוקת הפרויקט)

## מה זה הפרויקט
עורך וידאו אוטומטי מבוסס AI שרץ בטרמינל (CLI).
שני מצבים: (1) עריכת חומרי גלם קיימים, (2) יצירת סרטון מאפס מפרומפט.
השפה העיקרית: עברית. הממשק: CLI כרגע, GUI בעתיד.

## חוקי ברזל — לא לשבור לעולם

### 1. מודולריות מוחלטת
- כל יכולת (פיצ'ר) היא **step נפרד** בתיקייה `src/pipeline/steps/`
- כל step מייצא פונקציה אחת שמקבלת `StepContext` ומחזירה `StepResult`
- step **לעולם לא** מייבא מ-step אחר. תקשורת רק דרך ה-context
- step חדש = קובץ חדש. לא מוסיפים לוגיקה ל-step קיים

### 2. אל תיגע בליבה
- הקבצים הבאים הם **קריטיים**. לא לשנות בלי אישור מפורש:
  - `src/pipeline/engine.ts` (מנוע ה-pipeline)
  - `src/pipeline/types.ts` (טיפוסים משותפים)
  - `src/index.ts` (CLI entry point)
  - `src/utils/logger.ts` (מערכת לוגים)
- לפני כל שינוי בקבצים האלה: **הסבר מה אתה רוצה לשנות ולמה, וחכה לאישור**

### 3. שלב אחד בכל פעם
- לא לבנות 2 פיצ'רים במקביל
- לא להוסיף קוד "לעתיד" או "להכנה"
- כל PR/commit עושה דבר אחד בלבד
- אחרי כל שינוי — לבדוק שהפייפליין עובד מקצה לקצה

### 4. הגדרות בקונפיג, לא בקוד
- כל ערך שיכול להשתנות (threshold, path, model name) → ב-`config/default.json`
- **לא** hardcode ערכים בקוד

### 5. לוגים תמיד
- כל step חייב לכתוב ללוג: מה קיבל, מה עשה, כמה זמן לקח, מה החזיר
- שגיאות חייבות לכלול את ה-step שנכשל + הודעת שגיאה ברורה
- **לעולם לא** `console.log` — רק `logger.info()` / `logger.error()` / `logger.debug()`

### 6. טיפול בשגיאות
- כל step עטוף ב-try/catch
- step שנכשל **לא** מפיל את כל ה-pipeline — ממשיך לstep הבא (אלא אם הוגדר אחרת)
- שגיאות FFmpeg חייבות לכלול את הפקודה המלאה שנכשלה

### 7. קבצים זמניים
- כל קובץ זמני נוצר בתיקייה `temp/` בתוך תיקיית ה-output
- בסוף ה-pipeline — ניקוי אוטומטי של `temp/`
- שמות קבצים זמניים כוללים timestamp למניעת התנגשויות

### 8. עברית ו-RTL
- כל טקסט שמוצג למשתמש — בעברית
- שמות קבצים ומשתנים — באנגלית
- כתוביות — תמיד RTL, תמיד UTF-8

### 9. Git
- commit message באנגלית, קצר וברור
- פורמט: `feat: add X step` / `fix: resolve Y in Z step` / `refactor: simplify W`
- לא לעשות commit של node_modules, temp files, או קבצי וידאו

### 10. כשיש ספק — תשאל
- אם משימה לא ברורה → שאל לפני שאתה כותב קוד
- אם שינוי עלול להשפיע על steps אחרים → תציין את זה
- אם יש כמה דרכים לעשות משהו → הצג אותן ותן המלצה

## איך ה-Pipeline עובד

קובץ וידאו → [Step 1] → [Step 2] → [Step 3] → ... → קובץ תוצאה

- כל step מקבל `StepContext` שמכיל: נתיב קובץ נוכחי, metadata, הגדרות, logger
- כל step מחזיר `StepResult` עם: נתיב קובץ חדש (או אותו נתיב), metadata מעודכן, סטטוס
- המנוע (`engine.ts`) מריץ steps לפי הסדר שמוגדר ב-`config/default.json`
- step יכול להיות `enabled: false` בקונפיג → המנוע מדלג עליו

- **גל 5** תוצרים — versions, revisions, export, chat editor
- **גל 6** מתקדם — brand kit, templates, dubbing, face swap
- **גל 7** לימוד — self-improving system
